# 💍 Свадебное приглашение
> 🌐 **Вы можете протестировать сайт по**  [ссылке](https://na1x-dev.github.io/WeddingCard/)

Современный, полностью адаптивный одностраничный сайт-приглашение (свадебный лендинг). Включает в себя интерактивную карту, динамический таймер обратного отсчета (Rolling Clock), интерактивную смену цветовой палитры страницы и продвинутую RSVP-анкету с AJAX-отправкой данных на бэкенд Google Apps Script.

---

## ✨ Особенности проекта

* **Динамический дизайн и темы:** Гости могут интерактивно переключать цветовое оформление сайта (Песочный, Шампань, Оливковый, Небесно-голубой) с плавными переходами.
* **Анимированный таймер:** Кастомный счетчик времени до торжества с эффектом перелистывания цифр.
* **Умная RSVP-форма:** 
  * Поля анкеты динамически скрываются, если гость отмечает, что не сможет прийти.
  * Реализовано взаимоисключение в чекбоксах алкоголя (выбор «Не пью» автоматически снимает остальные галочки).
  * Защита от повторной отправки (кнопка блокируется и показывает встроенный CSS-спиннер во время запроса).
* **Двойной бэкенд (Sheets + Telegram):** Все ответы мгновенно сохраняются в Google Таблицу (с автоматической перезаписью при изменении ответа гостем) и дублируются мгновенным уведомлением в Telegram-чат.
* **Плавные анимации:** Появление контента при скролле через `IntersectionObserver` и высокая производительность за счет аппаратного сдвига каретки анимаций.

## 🚀 Инструкция по развертыванию под себя

### 1. Подготовка Telegram-бота
1. Создайте бота через [@BotFather](https://t.me/BotFather) и скопируйте его **токен**.
2. Создайте группу (или канал), добавьте туда бота и получите **Chat ID** (например, через бота `@userinfobot`).

### 2. Настройка базы данных и бэкенда (Google Таблицы)
1. Создайте новую **Google Таблицу**.
2. Перейдите в **Расширения** -> **Apps Script**.
3. Удалите стандартный код и вставьте содержимое файла `google_sheets.txt` вашего проекта.
4. В самом верху скрипта заполните ваши данные бота:
   ```javascript
   const TG_TOKEN = "ВАШ_ТОКЕН_БОТА";
   const TG_CHAT_ID = "ВАШ_ID_ЧАТА";
   ```
5. Нажмите кнопку **Начало развертывания** (Deploy) -> **Новое развертывание**.
6. Выберите тип: **Веб-приложение** (Web app).
7. В поле *Кто имеет доступ* (Who has access) **обязательно** укажите **Все** (Anyone).
8. Скопируйте предоставленный **URL веб-приложения**.

### 3. Привязка к сайту
1. Откройте файл `script.js`.
2. Вставьте скопированный URL в константу `WEB_APP_URL` на первой строчке:
   ```javascript
   const WEB_APP_URL = "ВАШ_URL_ВЕБ_ПРИЛОЖЕНИЯ";
   ```
3. *(Опционально)* Настройте дату свадьбы в переменной `targetDate` (по умолчанию установлена на 15 июня 2026 года).

### 4. Публикация на GitHub Pages
1. Загрузите файлы репозитория на свой аккаунт GitHub.
2. Перейдите в **Settings** -> **Pages** вашего репозитория.
3. В качестве источника выберите ветку `main` (или `master`) и нажмите **Save**.
4. Через пару минут сайт станет доступен по адресу `https://<ваш-логин>.github.io/<имя-репозитория>/`.
